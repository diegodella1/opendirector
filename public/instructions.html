OpenDirector

Sistema open-source de produccion de TV en vivo. Arma el rundown en la web, controla vMix desde cualquier lugar.

Que es OpenDirector

OpenDirector tiene dos partes que trabajan juntas:

Componente Que hace Donde corre
Webapp Browser Editor de rundown, teleprompter, panel de Go Live Cualquier browser (PC, tablet, celular)
Automator Windows Conecta al servidor y ejecuta comandos contra vMix La misma PC donde corre vMix

Como se conecta todo

Desde cualquier lugar: [Productor] ──HTTPS──> [Servidor] ──WebSocket──> [Automator + vMix] browser director.diegodella.ar PC del estudio [Talento] ──HTTPS──> [Servidor] prompter en browser El Automator habla con vMix por TCP local (127.0.0.1:8099). vMix nunca se expone a internet.

El servidor es el hub central. Todos se conectan a el. El Automator es el unico que habla con vMix, y siempre lo hace por red local.

Roles del equipo

Productor — Arma y dirige el show

  • Crea el show, arma bloques (segmentos), agrega elements (clips, lower thirds, graficos)
  • Configura GT Templates para los lower thirds de vMix
  • Escribe el script del teleprompter por bloque
  • Sube archivos de media (videos, imagenes)
  • En vivo: ve el estado, edita textos de LT al vuelo, envia senales al talento

Operador — Ejecuta contra vMix

  • Instala el Automator en la PC con vMix
  • Conecta al servidor y selecciona el show
  • Ve el rundown con todos los bloques y elements
  • Aprieta CUE para ejecutar cada element — el Automator envia los comandos a vMix
  • La media se descarga automaticamente a la PC local

Talento — Lee el teleprompter

  • Abre el link del prompter en un browser (puede ser tablet, celular, PC)
  • Ve el script del bloque actual con scroll automatico
  • Recibe senales del productor (countdown, wrap, stretch, go)

Descargar Automator para Windows

Instala el Automator en la PC donde corre vMix.
Requiere Windows 10/11 (64-bit).

Descargar Automator (MSI)

Ir a GitHub Actions > ultimo build exitoso > descargar el artifact automator-*.msi

Setup del Operador (paso a paso)

  1. Instalar el Automator
    Descargar el MSI desde GitHub Actions e instalarlo. Se instala como app de escritorio.
  2. Preparar vMix
    Asegurarse de que vMix este corriendo y que el API TCP este habilitado (Settings > Web Controller > habilitar).
    Crear los inputs GT Title que se van a usar (ej: LT_STANDARD con campos Headline.Text y Description.Text).
  3. Abrir el Automator y conectar al servidor
    Server URL: https://director.diegodella.ar
    El Automator carga la lista de shows disponibles. Seleccionar el show.
  4. Conectar a vMix
    vMix Host: 127.0.0.1 (localhost, porque esta en la misma PC).
    vMix Port: 8099 (default de vMix).
  5. Verificar media sync
    Los archivos de media del show se descargan automaticamente a C:\OpenDirector\Media.
    El panel de Media Sync muestra el progreso. Verificar que todo este en "synced".
  6. Operar el show
    Navegar los bloques con los botones de Next/Prev.
    Hacer click en CUE para ejecutar cada element. Los lower thirds con GT Template se envian automaticamente (SetText + OverlayIn).

GT Templates (Lower Thirds configurables)

Los GT Templates conectan los lower thirds de la webapp con los GT Title inputs de vMix.

Como funciona

  1. El productor crea un GT Template en la webapp (ej: "Nombre+Cargo")
  2. Configura: Input Key de vMix (LT_STANDARD), numero de overlay, y los campos (ej: Headline.Text = "Nombre", Description.Text = "Cargo")
  3. En cada bloque, agrega un element Lower Third y selecciona el GT Template
  4. Escribe los valores (ej: Nombre = "Diego Della", Cargo = "Productor")
  5. Cuando el operador hace CUE, el Automator envia a vMix:
> FUNCTION SetText Input=LT_STANDARD&SelectedName=Headline.Text&Value=Diego Della > FUNCTION SetText Input=LT_STANDARD&SelectedName=Description.Text&Value=Productor > FUNCTION OverlayInput2In Input=LT_STANDARD

Requisitos en vMix

  • El input debe ser un GT Title (no un input comun)
  • El Key del input debe coincidir con el que se configuro en la webapp
  • Los nombres de campo deben ser exactos (ej: Headline.Text, no headline)
  • Para ver los nombres de campo: en vMix, click derecho en el input GT > Title Editor > ver los nombres de cada text block

Preguntas frecuentes

Necesito exponer vMix a internet?

No. El Automator corre en la misma PC que vMix y se conecta por localhost. Solo el servidor web (Raspberry Pi) esta expuesto via Cloudflare Tunnel.

Puedo tener el estudio en otro pais?

Si. El productor puede estar en Argentina, el estudio con vMix en USA, y el talento en otro lugar. Todos se conectan al servidor por internet.

Que pasa si se cae internet?

El Automator tiene cache local del rundown. Puede seguir operando contra vMix sin conexion al servidor. Cuando vuelve internet, se re-sincroniza.

Puedo usar multiples estilos de lower third?

Si. Crea un GT Template por cada estilo. Cada uno apunta a un input GT diferente en vMix con sus propios campos.

Como actualizo los textos en vivo?

El productor puede editar los campos de texto de un lower third durante el show en vivo. El cambio se sincroniza por WebSocket. El proximo CUE del operador usa los valores nuevos.

Before You Start

Make sure the entire environment is up and running before starting the tests.

ⓘ
The OpenDirector server must be running. The Automator is tested from a Windows PC running vMix. The prompter can be tested from any device with a browser.

Requirements

  • OpenDirector server running and accessible
  • PC with vMix open and at least 3 inputs loaded
  • Automator installed on the vMix PC
  • A second device (tablet/phone/another PC) for the prompter
  • At least 1 short video clip (10-30 sec) to upload as media
  • All devices on the same local network

System URLs

Replace SERVER with the server IP or hostname (e.g. 192.168.1.14:3000).

Role Overview

Producer

Builds the rundown in the browser, uploads media, writes scripts, configures the show and monitors the live output from Go Live. Sends signals to talent.

Operator

Uses the Automator (Windows app) to execute the rundown against vMix. Advances blocks, fires elements, controls clips and graphics.

Host / Talent

Reads the script from the teleprompter in a browser. Receives visual signals from the producer (countdown, wrap, go). No installation needed.

Producer — Producer Tests

Open the browser and navigate to http://SERVER/
Where: Any PC/Mac/tablet with a modern browser

1. Create Show

PRE-SHOW
  • Go to Home (/) and verify the show list loads
  • Click "New Show" → a blank show is created and the editor opens
  • Verify the new show appears with Draft status
  • Rename the show from the editor (click on the title)
  • Go back to Home and confirm the name has updated
✓
If there are saved templates, also try creating a show from a template.

2. Build Rundown

PRE-SHOW
  • Add at least 3 blocks to the rundown
  • Name each block (e.g. "Opening", "Segment 1", "Closing")
  • Set an estimated duration on each block (e.g. 120 sec, 180 sec, 60 sec)
  • Drag & drop: reorder blocks by dragging → verify the order is saved
  • Write a prompter script in at least 1 block (Script field)
  • Use special markup in the script: [PAUSA], [VTR: Test Clip], **bold text**, (instruction)
  • Add production notes in at least 1 block
  • Assign cameras to blocks (e.g. CAM1, CAM2)
  • Test Undo (Ctrl+Z) after a change → the change is reverted
  • Test Redo (Ctrl+Y) → the change is re-applied

3. Elements & Actions

PRE-SHOW

Inside the blocks, add different types of elements:

  • Add a CLIP element → assign a media file to it (uploaded in the Media step)
  • Add a GFX (graphic) element → configure vMix input key and overlay
  • Add a LT (lower third) element → assign a GT Template and fill in the text fields
  • Add an AUD (audio) element → configure the audio function (AudioOn, SetVolume, etc.)
  • Add a NOTE element → verify it is informational only, does not execute actions
  • Configure actions on at least 1 element: on_cue (when CUE'd), step (manual button), on_exit (when leaving)
  • Reorder elements within a block using drag & drop
  • Delete an element → confirm it disappears
⚠
vMix inputs are referenced by Key (unique name), never by number. Verify that the keys match what is configured in vMix.

4. GT Templates (Lower Thirds)

PRE-SHOW
  • Create at least 1 GT Template in the show configuration
  • Configure: vMix input key of the GT Title, overlay number, field names (e.g. Headline.Text, Description.Text)
  • Assign the GT Template to an LT element
  • Fill in the text values on the LT element
  • Verify that the lower third preview shows the correct values

5. Media

PRE-SHOW
  • Upload at least 1 video file (drag & drop or upload button)
  • Verify a thumbnail is generated automatically
  • Verify file info appears (codec, duration, resolution)
  • If the codec is not vMix-compatible, verify a warning appears (does not block)
  • Assign the media to a CLIP element in the rundown

6. Prompter Configuration

PRE-SHOW
  • Open the prompter configuration in the show editor
  • Adjust font size, font family, colors
  • Configure guide line position (default: 33%)
  • Configure default scroll speed
  • Verify changes are saved

7. Set to Ready

PRE-SHOW
  • From the editor, change the show status from Draft to Ready
  • Verify the status badge updates
  • Confirm the show is still editable in Ready state
  • Verify the show now appears available for the Automator
ⓘ
The Automator can only connect to shows in Ready, Rehearsal or Live state. It cannot connect to Draft shows.

Operator — Operator Tests (Automator)

Open the Automator on the PC running vMix
Where: Windows PC with vMix open

1. Connection

PRE-SHOW
  • Open the Automator → the connection screen appears
  • Enter the server URL (e.g. http://192.168.1.14:3000)
  • Verify the show dropdown loads available shows (only Ready/Rehearsal/Live)
  • Select the test show
  • Configure vMix Host (default: 127.0.0.1) and Port (default: 8099)
  • Configure the media folder (e.g. C:\OpenDirector\Media)
  • Click CONNECT
  • Verify green WebSocket connected indicator in the status bar
  • Verify green vMix connected indicator in the status bar
  • Verify the full rundown loads in the main panel
⚠
If the vMix connection fails, verify that vMix is open and the TCP API is enabled (Settings → Web Controller → Enable TCP API).

2. Pre-flight Check

PRE-SHOW
  • Open the Pre-flight panel
  • Click "Run Check"
  • Verify vMix inputs are validated (clip pools, GT Titles, etc.)
  • If there are errors (red X), fix the configuration and re-check
  • If there are warnings (yellow !), review but fixing is not mandatory
  • Achieve at least all checks at OK (green checkmark) or warning

3. Media Sync

PRE-SHOW
  • Verify the Media Sync panel shows the files to download
  • Wait for all files to download (progress bars)
  • Verify each file shows "synced" status (green)
  • Verify checksums (SHA256) match
  • Confirm files are in the configured media folder

4. Rehearsal

LIVE

Before going on air, do a full rehearsal run:

  • The producer sets the show to Rehearsal (or the operator with Ctrl+R)
  • Verify the status badge shows "REHEARSAL" (yellow)
  • Press Space to advance to the first block
  • Verify the current block is highlighted (red left border)
  • Select an element with arrow keys ↑/↓ or click
  • Press Enter to CUE the selected element
  • Verify the action executes in vMix (the video/graphic appears)
  • Test step buttons (F1-F8) if step actions are configured
  • Advance through all blocks with Space
  • Verify timers: show elapsed, block elapsed, block remaining
  • Verify back-times on future blocks (projected time in blue)
  • Verify tally: PGM (red) and PVW (green) in the tally panel
  • Press Esc to STOP → verify the show stops
  • Press R to RESET → verify it returns to the beginning
✓
In Rehearsal, commands are sent to vMix for real. Use it to validate everything is properly configured before going live.

5. On Air (Live)

LIVE
  • The producer sets the show to Live (Go Live button or F5 from Go Live panel)
  • Verify the status badge shows "LIVE" (red, animated)
  • Execute the full show: advance blocks, cue elements
  • Verify clips play correctly (progress bar with position)
  • Verify A/B clip pool alternation (indicator changes between POOL A and POOL B)
  • Verify lower thirds display with correct text
  • Verify over/under timers appear when blocks complete (+0:32 red = overrun, -0:15 green = under)
  • Test AUTO mode (toggle in status bar): elements with trigger on_block_start fire automatically
  • Verify the execution log: each command shows timestamp, vMix response and latency

6. Panic & Recovery

LIVE
  • Press F12 (PANIC) → verify it cuts to a safe input (requires overrun_safe_input_key configured)
  • Simulate server disconnection (briefly turn off WiFi on the Automator) → verify "OFFLINE" banner appears
  • Reconnect → verify the Automator reconnects and syncs
  • Verify the show keeps running during disconnection (uses local SQLite cache)

Host / Talent — Prompter Tests

Open http://SERVER/shows/{id}/prompter in a browser
Where: Tablet, phone, laptop or dedicated monitor

1. Open the Prompter

PRE-SHOW
  • Navigate to http://SERVER/shows/{id}/prompter
  • Verify the full script loads (all blocks concatenated)
  • Verify black background with white text (or the configured colors)
  • Verify the red guide line is visible horizontally (indicates reading position)
  • Verify block names appear as gray section headers
  • Verify rendered markup:
    • [PAUSA] → centered yellow separator
    • [VTR: name] → block with purple left border
    • **text** → white bold
    • (instruction) → gray italic at reduced size

2. Prompter Controls

PRE-SHOW
  • Press Space → toggles auto-scroll on/off
  • With auto-scroll on, press ↓ → increases speed
  • Press ↑ → decreases speed
  • Press + → increases font size
  • Press - → decreases font size
  • Press F → enters fullscreen
  • Press F again → exits fullscreen
  • Press M → enables mirror mode (text flipped horizontally, for glass prompters)
  • Press M again → disables mirror mode
  • Press C → shows the configuration panel with sliders
  • In the config panel: adjust font size, scroll speed, guide position, margin

3. Producer Signals

LIVE

While the show is in Rehearsal or Live, the producer sends signals from the Go Live panel. The host sees them as large overlays at the top of the prompter.

ⓘ
For this test, you need someone (or yourself in another window) on the Go Live panel (/shows/{id}/live) to send the signals.
  • Countdown 30s: yellow overlay appears with countdown from "30" to "0", then disappears
  • Countdown 1min: yellow overlay with 60-second countdown
  • WRAP: blinking red overlay, stays until the producer clears it
  • STRETCH: blue "STRETCH" overlay, disappears after 10 seconds
  • STANDBY: yellow "STANDBY" overlay, disappears after 10 seconds
  • GO: green "GO" overlay, disappears after 3 seconds
  • Custom: purple overlay with free text typed by the producer, 10 seconds

4. Scroll Sync

PRE-SHOW

If there are 2 devices with the prompter open, scrolling can be synchronized between them.

  • Open the prompter on 2 different devices for the same show
  • On device 1, press C and set scroll sync to MASTER
  • On device 2, press C and set scroll sync to FOLLOWER
  • Scroll on the MASTER device → verify the FOLLOWER follows
  • Change speed on MASTER → verify FOLLOWER matches

5. Offline Mode

PRE-SHOW
  • With the prompter open and loaded, disconnect the device from the network
  • Verify a yellow banner appears: "OFFLINE — Using cached scripts"
  • Verify auto-scroll, keyboard and touch controls still work
  • Verify mirror mode still works
  • Reconnect to the network → verify the banner disappears and it reconnects
⚠
In offline mode, producer signals and live script edits do not work. Only scroll and local controls.

Integration Test — Full Flow

This test simulates a real show from start to finish. Requires all 3 roles active simultaneously.

ⓘ
If only 1 person is available, open 3 windows/devices: one as producer (Go Live), one with the Automator, and one with the prompter.
  1. Producer: Verify the show is Ready and the rundown is complete Open /shows/{id}/live (Go Live panel). Verify all blocks are visible in the left list.
  2. Operator: Connect the Automator to the show Verify connection to server and vMix. Run pre-flight. Confirm media is synced.
  3. Host: Open the prompter Navigate to /shows/{id}/prompter. Press F for fullscreen. Enable auto-scroll with Space.
  4. Producer: Switch to Rehearsal From Go Live, activate REHEARSAL. Verify the Automator and prompter reflect the status change.
  5. Operator: Do a quick rehearsal Press Space to advance to block 1. Cue a couple of elements. Verify vMix responds. STOP + RESET.
  6. Producer: Go Live Press F5 or the "GO LIVE" button. Verify the red "LIVE" badge on all panels.
  7. Operator: Execute block 1 Space to start. CUE the elements in the block. Verify clips, graphics and lower thirds in vMix.
  8. Producer: Send a signal to talent Send a 30-second countdown. Verify the host sees it in the prompter.
  9. Operator: Advance to block 2 Space. Verify the previous block is marked as "done" with an over/under badge.
  10. Producer: Monitor timers Verify show elapsed, block elapsed, block remaining. Verify back-times on future blocks.
  11. Producer: Send WRAP to talent Send a WRAP signal. Verify the blinking red overlay on the prompter. Clear the WRAP.
  12. Operator: Complete the show Advance through all remaining blocks. Execute at least 1 element per block.
  13. Producer: Stop the show Press STOP or Esc. Verify it returns to Ready state. Review the execution log.

Edge Cases

Optional tests for edge scenarios:

  • Live editing: while the show is Live, edit the script of a future block from the editor → verify the prompter updates the text
  • Live editing (GT fields): change the text of a lower third on a future block → verify the new text appears when CUE'd
  • Automator disconnection: close the Automator during a live show → verify the producer sees the disconnection in Go Live
  • Automator reconnection: reconnect the Automator → verify it syncs state and rundown
  • Multiple tabs: in the Automator, open a second show in another tab ("+" button) → verify each tab operates independently
  • Template: save the show as a template from Home → create a new show from that template → verify all blocks and elements are copied (without media)
  • Block overrun: let a block run past its estimated duration → verify the timer changes from green to red and the badge shows +X:XX
  • Archive: set the show from Ready to Archived → verify it becomes read-only and the Automator cannot connect
